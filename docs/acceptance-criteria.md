# Acceptance Criteria – PhishGuard

## Purpose
Acceptance criteria define the conditions required for a user story to be considered complete.

---

## US-001
### Submit Suspicious Email Content

**Acceptance Criteria:**
- User can input suspicious message content
- Submission field accepts text input
- System acknowledges submission
- System processes submitted content

---

## US-002
### Submit Suspicious URLs

**Acceptance Criteria:**
- User can paste URL input
- URL format is accepted
- System identifies suspicious URL patterns
- System processes URL analysis request

---

## US-003
### Assign Phishing Risk Level

**Acceptance Criteria:**
- System returns a clear risk classification
- Risk levels include:
  - Low
  - Medium
  - High
  - Critical
- Risk level is displayed clearly

---

## US-004
### Explain Suspicious Indicators

**Acceptance Criteria:**
- System identifies at least one phishing indicator
- Indicators are presented clearly
- Explanations are understandable to non-technical users

Example indicators:
- Suspicious domain
- Urgent language
- URL shortening
- Spoofing indicators

---

## US-005
### Provide Safety Recommendations

**Acceptance Criteria:**
- System provides practical next-step guidance
- Recommendations are clearly written
- Advice is understandable to general users

Examples:
- Do not click suspicious links
- Verify sender identity
- Report suspicious communication
- Delete fraudulent message

---

## US-008
### User-Friendly Interface

**Acceptance Criteria:**
- Interface is simple and easy to navigate
- Core functionality is clearly visible
- Non-technical users can understand system outputs
