# User Stories – PhishGuard

## User Story Format
All user stories follow the Agile format:

**As a [user], I want [goal] so that [benefit].**

---

## Core User Stories

### US-001
**As a user, I want to submit suspicious email content so that I can determine whether it may be phishing.**

Priority: High

---

### US-002
**As a user, I want to submit suspicious URLs so that I can determine whether they may be unsafe.**

Priority: High

---

### US-003
**As a user, I want the system to assign a phishing risk level so that I can understand the potential threat severity.**

Priority: High

---

### US-004
**As a user, I want explanations for flagged phishing indicators so that I understand why content is considered suspicious.**

Priority: High

---

### US-005
**As a user, I want practical cybersecurity safety recommendations so that I know what action to take next.**

Priority: High

---

## Optional / Future User Stories

### US-006
**As a user, I want to view previous submissions so that I can review past phishing checks.**

Priority: Medium

---

### US-007
**As an administrator, I want to view suspicious submission trends so that phishing patterns can be analyzed.**

Priority: Low

---

### US-008
**As a user, I want a simple and intuitive interface so that I can use the platform without technical expertise.**

Priority: High

---

### US-009
**As a future organization user, I want reporting features so that phishing awareness metrics can be reviewed.**

Priority: Low
