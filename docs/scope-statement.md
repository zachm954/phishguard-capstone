# Scope Statement – PhishGuard

## Project Scope Overview
PhishGuard is being developed as a cybersecurity-focused web application concept for FIU Capstone 1.

The project will focus primarily on Agile/Scrum execution, cybersecurity planning, documentation, requirements gathering, design artifacts, and optional lightweight prototyping.

Capstone 1 will emphasize project management and product planning rather than production-level implementation.

---

## In Scope for Capstone 1

### Project Planning
- Product vision definition
- Problem statement development
- Scope definition
- Stakeholder identification
- Scrum role assignment
- Sprint planning
- Product backlog creation
- Sprint backlog creation

### Requirements Engineering
- User stories
- Acceptance criteria
- Feature prioritization
- Functional requirements
- Non-functional requirements

### Cybersecurity Analysis
- Risk assessment
- Threat identification
- Vulnerability analysis
- Security requirements definition
- Basic phishing indicator research

### Design Artifacts
- UI wireframes
- Workflow diagrams
- Architecture diagrams
- System design concepts

### Scrum Deliverables
- Meeting minutes
- Sprint notes
- Sprint reviews
- Sprint retrospectives
- Team coordination documentation

### Optional Work
If team bandwidth allows:

- Low-fidelity clickable prototype
- Basic UI mockup
- Simple concept demonstration

---

## Out of Scope for Capstone 1

The following items are intentionally excluded from required Capstone 1 scope:

- Production-ready deployment
- Full backend implementation
- Live phishing URL scanning
- Real-time threat intelligence integrations
- Browser extension development
- Email inbox integrations
- Malware sandbox analysis
- Machine learning implementation
- Enterprise-grade authentication systems
- Production security hardening
